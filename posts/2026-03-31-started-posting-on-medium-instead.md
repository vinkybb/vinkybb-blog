# (started posting on Medium instead)

# 从 GitHub Pages 到 Medium：博客平台迁移的思考

## 背景介绍

在技术写作领域，Andrej Karpathy 的博客堪称标杆。这位深度学习领域的知名研究者、前 Tesla AI 负责人，曾以其清晰的技术讲解和独特的视角吸引了大量读者。然而，细心的读者会发现，他的博客在很长一段时间里处于沉默状态——最后一次更新已是两年有余。

这并非作者离开了技术写作领域，而是发生了一个值得关注的变化：**平台迁移**。在2018年1月20日，Karpathy 发布了一篇简短的说明文章，标题直白地写道"started posting on Medium instead"（已经开始在 Medium 发布）。这个看似简单的决定，背后折射出技术内容创作者在平台选择上的深层思考。

## 核心内容：迁移的原因与策略

### 时间与精力的现实约束

Karpathy 在文中坦诚地解释了博客长期停更的核心原因：**加入 Tesla 后，工作占据了大量的时间和精力**。这不是一个借口，而是技术从业者的真实写照。在高强度的工作环境下，维护一个高质量的技术博客确实需要付出巨大的心智成本——选题、研究、写作、编辑、代码验证，每一个环节都不能敷衍。

对于技术博客而言，"高质量"的标准往往比普通博客更高。读者期待的是深入的分析、可复现的实验、精确的代码示例。这种标准背后，是大量的时间投入。当工作本身已经消耗了大部分精力时，博客的自然降温几乎是必然的。

### 为什么选择 Medium？

Karpathy 选择 Medium 作为新的发布平台，核心原因是**更快速便捷的发布体验**。这个理由看似简单，却蕴含着深刻的产品洞察。

传统自建博客（如 GitHub Pages）虽然提供了完全的控制权，但发布流程往往较为繁琐：
- 内容需要本地编写并格式化
- 通过 Git 推送到仓库
- 等待静态站点生成
- 可能需要调整样式或修复部署问题

相比之下，Medium 提供了一个"开箱即用"的写作环境：
- 简洁的富文本编辑器
- 自动化的排版和样式
- 即时发布，无需技术配置
- 内置的读者群体和分发机制

对于已经疲惫的技术从业者而言，Medium 的便捷性降低了"开始写作"的门槛。**减少阻力，本身就是一种激励**。

### 双轨策略的务实选择

Karpathy 并没有完全放弃原有博客。他采用了务实的双轨策略：**短到中等长度的内容优先发布在 Medium，而长篇幅的深度文章可能仍会回到原博客**。

这种策略体现了一种平台意识的成熟：
- **短内容**更适合快速发布平台，读者期望快速获取信息，Medium 的阅读体验更匹配
- **长内容**需要更沉浸的阅读环境，自建博客可以提供更好的控制权（如代码高亮、数学公式、自定义交互元素）

这不是"放弃"，而是"适配"。不同的内容类型，选择最适合的平台。

## 技术分析：博客平台的演变趋势

### 从自建到托管：趋势的转变

Karpathy 的迁移反映了一个更广泛的趋势：**技术内容创作者从自建平台转向托管平台**。

在早期（2010-2015），GitHub Pages、Jekyll、Hexo 等自建博客方案在技术社区风靡一时。开发者享受着"自己掌控一切"的快感：自定义域名、个性化主题、完整的代码示例、无限制的内容结构。

然而，随着时间推移，许多人发现：**维护本身的成本，开始超过了掌控带来的收益**。Git 操作、主题更新、插件兼容、部署调试——这些技术细节开始消耗写作的心力。

Medium、Substack 等托管平台的兴起，恰恰提供了一个"减负"的选择。虽然失去了部分控制权，但换来的是更纯粹的创作体验。

### 发布体验的重要性

Karpathy 的选择揭示了一个容易被忽视的因素：**发布体验直接影响创作频率**。

这不是一个理论推测，而是心理学中的"阻力效应"：当一个行为的阻力增加，该行为的发生频率会显著下降。对于博客写作而言：

- 如果每次发布需要30分钟的配置工作，作者可能会推迟写作
- 如果发布只需要5分钟的编辑和点击，作者更容易保持活跃

Medium 的设计理念正是减少阻力：编辑器简洁流畅，发布一键完成，排版自动优化。这种体验设计，无形中成为了创作的催化剂。

### 内容分发与读者触达

除了发布体验，Medium 还提供了另一个自建博客难以匹敌的优势：**内置的读者群体和分发机制**。

自建博客依赖作者的主动推广（社交媒体、邮件列表、SEO优化）。Medium 则拥有：
- 平台内部的推荐算法
- 已有的读者群体
- 社交分享的便捷性
- 订阅和关注机制

对于时间有限的技术从业者，这种"自带流量"的特性，降低了运营成本，让内容更容易触达目标读者。

### 双轨策略的深层逻辑

Karpathy 的双轨策略并非妥协，而是**基于内容属性的精准匹配**：

**短内容（Medium）的特征**：
- 快速阅读（5-15分钟）
- 单一核心观点
- 较少代码或复杂示例
- 讨论性或思考性为主

**长内容（自建博客）的特征**：
- 深度阅读（30分钟以上）
- 多个技术细节
- 大量代码示例或实验数据
- 教学性或实证性为主

这种策略避免了"一刀切"，而是让平台服务于内容，而非内容适应平台。

## 总结

Karpathy 的博客迁移，不是一个简单的"换平台"决定，而是技术内容创作者在现实约束下做出的理性调整。

核心启示：

1. **现实约束需要务实应对**：时间和精力是有限资源，平台选择应该适配现实状态
2. **发布体验影响创作频率**：减少阻力，本身就是对创作的支持
3. **内容与平台的匹配**：不同内容类型适合不同平台，双轨策略比单一选择更灵活
4. **控制权与便捷性的权衡**：自建平台提供控制，托管平台提供便捷，选择取决于当下需求

技术博客的价值不在于平台的完美，而在于内容的质量。Karpathy 的选择告诉我们：**选择合适的工具，让创作本身成为焦点**。对于每一位技术内容创作者，这或许是最值得思考的命题。
