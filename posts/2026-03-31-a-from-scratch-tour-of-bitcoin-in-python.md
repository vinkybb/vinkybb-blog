# 用 Python 从零实现比特币

区块链最具革命性的创新在于将**开源软件开发**延伸为**开源 + 状态**。我们不仅共享代码，更共享一台"运行中的计算机"——任何人都可以开放、无许可地使用它。这场范式革命的种子，始于比特币。

为了真正理解比特币的工作原理，我践行 Feynman 的理念："我不能创造的，我就不理解"。本文将用纯 Python、零依赖的方式，从零实现比特币的核心功能：创建交易、数字签名、并广播到网络。

## 背景与动机

比特币使用椭圆曲线密码学（ECC）而非常见的 RSA 来保障交易安全。选择 ECC 的原因在于其更高的安全性密度——相同安全级别下，ECC 的密钥长度远小于 RSA。比特币采用 secp256k1 曲线，这条曲线由三个整数参数定义：p（模数）、a、b。有趣的是，NIST 发布了推荐曲线列表，但比特币社区选择使用更不可能含有后门的曲线。

## 核心实现步骤

### 第一步：生成密码身份

密码身份即私钥-公钥对。私钥是一个随机整数（1 ≤ key < n），而公钥是椭圆曲线上的一个点，通过将生成点 G 加到自身 secret_key 次得到：

```python
# 椭圆曲线定义
bitcoin_curve = Curve(p=..., a=0, b=7)
G = Point(bitcoin_curve, x=..., y=...)

# 私钥生成
secret_key = random.randrange(1, bitcoin_gen.n)

# 公钥计算：使用"double and add"算法加速
public_key = secret_key * G
```

关键在于椭圆曲线加法的特殊定义。曲线上两点相加的结果仍是曲线上的点，这需要实现扩展欧几里得算法和模乘逆元。看起来复杂，但本质上只是对坐标 (x,y) 进行一系列加乘运算，中间穿插模 p 操作。

### 第二步：生成钱包地址

比特币地址并非公钥本身，而是从公钥确定性派生而来，包含额外特性（如内嵌校验和）。地址生成需要两个哈希函数：SHA-256 和 RIPEMD-160。

作为"零依赖"实现，我按照 NIST FIPS PUB 180-4 文档，用纯 Python 实现了 SHA-256。这个算法涉及一系列位运算（右旋、移位、异或等）和64个常量（取自前64个素数立方根的小数部分前32位）。

地址生成流程：公钥 → SHA-256 → RIPEMD-160 → Base58Check 编码（带校验和）。校验和机制防止转录错误，确保地址有效性。

### 第三步：构建交易

比特币交易基于 UTXO（未花费交易输出）模型。交易结构包含版本号、输入列表、输出列表和锁定时间。每个输入引用前一笔交易的输出，每个输出指定金额和接收地址。

交易签名使用 ECDSA（椭圆曲线数字签名算法）。签名过程涉及生成随机 nonce k，计算曲线点 R = k*G，并派生签名值 s = k⁻¹(z + r*secret_key)，其中 z 是交易哈希。

广播交易需要连接比特币网络的节点。我实现了简化的网络协议，构造符合比特币规范的交易消息，通过 TCP 连接发送到节点。

## 技术深度分析

这个实现路径的价值在于**逐层构建而非抽象调用**。通过亲手实现椭圆曲线运算，我深刻理解了：

- **私钥本质**：随机整数，但掌控所有关联资金
- **签名原理**：数学关系确保签名不可伪造，且签名者身份可验证
- **交易结构**：UTXO 模型不同于账户模型，提供更强的隐私和并发性
- **网络协议**：比特币节点间通过 P2P 网络传播交易和区块

代码虽看似"复杂"（模运算、位操作、编码转换），但每一步都有清晰的数学依据。不依赖抽象库，反而迫使你理解每个细节的技术必要性。

## 总结

从零实现比特币是一次极佳的技术学习之旅。这不仅是代码编写，更是对密码学、分布式系统、网络协议的深度探索。通过"创造来理解"，我们超越了表面知识，获得了对区块链底层原理的直觉把握。

这个项目适合想深入理解区块链的开发者。技术门槛中等——需要 Python 编程能力和基础密码学概念。但回报丰厚：你将真正理解比特币如何表示和传递价值，以及为何它能成为信任机器的基石。
