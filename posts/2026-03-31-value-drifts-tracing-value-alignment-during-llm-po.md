# Value Drifts: Tracing Value Alignment During LLM Post-Training

# Value Drifts：追踪大语言模型后训练过程中的价值对齐变化

## 背景：从静态评估到动态追踪的转变

大语言模型的价值对齐研究近年来备受关注，但主流方法存在一个根本性局限——它们几乎都采用静态评估视角。研究者们通常训练完成模型后，再使用各种基准测试检验模型是否符合人类价值观，这种"事后检验"的方式忽略了一个关键问题：模型的价值取向究竟是如何在训练过程中逐步形成的？

这项发表在 arXiv 上的研究填补了这一空白，首次将研究视角从静态评估转向动态追踪，系统性探究了模型在后训练各个阶段如何学习和内化人类价值观。这一范式转换的意义在于，它帮助研究者理解价值对齐的形成机制，从而在训练过程中主动干预，而非事后补救。

## 核心发现：监督微调是价值观形成的关键窗口

研究团队通过大量实验揭示了一个重要规律：**监督微调（SFT）阶段是模型价值观形成的关键期**。在这一阶段，模型建立起核心的价值取向框架，而后续的偏好优化阶段（无论是 RLHF 还是 DPO 等方法）往往难以显著调整这些已形成的价值观。

这一发现具有深刻的实践意义。传统观点认为，偏好优化是实现价值对齐的核心环节，大量研究和工程资源投入到偏好数据的收集和优化算法的设计上。然而，这项研究提示我们：如果在 SFT 阶段没有奠定正确的价值基础，后续的偏好优化可能只是"在错误的地基上装修"。开发者应当重新审视 SFT 数据的价值取向，而非仅仅关注偏好数据的质量。

## 算法选择本身影响价值形成

研究还发现另一个容易被忽视的因素：即使使用完全相同的偏好数据，采用不同的优化算法也会产生不同的价值对齐结果。这意味着算法选择并非纯粹的技术决策，它本身会影响模型价值取向的形成。

具体而言，RLHF、DPO、SimPO 等不同偏好优化算法在数学原理和优化目标上存在差异，这些差异会反映到最终的模型行为中。研究团队通过解耦算法与数据集的影响，测量了价值漂移的幅度和时间特征，发现算法之间的差异在某些价值维度上甚至比数据差异更显著。

这一发现提醒实践者：在设计和选择优化算法时，不仅要考虑收敛速度、计算效率等传统指标，还需要审视算法本身可能引入的价值偏好。算法设计应当成为价值对齐工程的一部分，而非仅仅是工具选择。

## 研究方法：可控的价值操纵实验

为了实现动态追踪，研究团队设计了一套创新性的实验框架。他们选择 Llama-3 和 Qwen-3 等主流模型家族的不同尺寸版本，使用合成偏好数据集实现可控的价值操纵。所谓"可控"，是指研究者可以精确控制数据中蕴含的价值取向，从而观察模型在特定价值刺激下的响应模式。

研究团队采用解耦设计：分别控制后训练算法和数据集，独立测量两者对价值漂移的影响。这种设计类似于心理学实验中的变量控制，能够剥离复杂因素，揭示单一变量的真实作用。通过测量价值漂移的幅度（模型价值取向变化的大小）和时间（变化发生在训练的哪个阶段），研究者绘制出了价值形成的动态轨迹。

## 实践启示：数据筛选与算法选型的新视角

这项研究为工业界的 LLM 训练实践提供了可操作的指导建议。在数据筛选层面，开发者应当意识到 SFT 数据的价值取向比偏好数据更关键。如果 SFT 数据中存在系统性偏差，后续的偏好优化很难纠正。因此，SFT 数据的筛选标准需要纳入价值维度，而非仅仅关注任务相关性或质量指标。

在模型选择层面，不同模型家族的预训练基础会影响 SFT 阶段的价值敏感性。研究者观察到 Llama-3 和 Qwen-3 在某些价值维度上表现出不同的漂移模式，这提示开发者在选择基座模型时也需要考虑其潜在的价值倾向。

在算法选型层面，偏好优化算法的选择应当结合价值目标进行考量。如果希望模型保持某种特定的价值取向，选择算法时需要考虑该算法在相关价值维度上的历史表现，而非仅仅追求性能指标。

## 总结与展望

这项研究将 LLM 价值对齐从"结果导向"推向"过程导向"，揭示了训练过程中价值形成的关键节点和影响因素。SFT 阶段的关键性、算法选择的价值影响这两个发现，为实践者提供了新的干预窗口和决策依据。

未来的研究可以进一步细化价值维度的分类，探索不同价值类型在训练过程中的差异化形成模式。此外，将这套追踪框架扩展到多模态模型和多轮对话场景，也将为价值对齐研究带来新的洞察。随着 LLM 在社会各领域的深入应用，理解并主动塑造其价值取向，将是负责任 AI 开发的重要组成部分。
