# Show HN: Skillwave – agent orchestrator with async comms and goal-alignment loop

# Skillwave：基于异步通信与目标对齐循环的 Agent 编排器

## 背景：自主 Agent 系统的编排困境

随着大语言模型能力的持续提升，构建能够自主完成端到端项目的 agent 系统成为技术社区的热点方向。然而，当前的 agent 编排方案普遍面临三大挑战：长期目标的一致性难以维持、用户交互阻塞系统执行、以及复杂工作流的状态管理成本高昂。传统方案往往依赖数据库存储状态、通过同步对话获取用户输入，导致系统架构复杂、迁移成本高、且容易在长时间运行中偏离初始目标。

在这一背景下，Skillwave 提出了一个轻量却强大的编排范式——通过 Markdown 文件作为持久真相源、异步文件处理用户沟通、Git 提供可追溯版本历史，构建了一个无需数据库依赖的自主 agent 工作流脚手架。

## 核心内容：从用户故事到自主执行

Skillwave 的核心定位是一个"种子仓库"而非框架或库，其工作流程清晰明确：用户提供一份包含 50-100 个用户故事的 `GOAL.md` 文件，系统自动将工作分解为具体任务、动态创建适配的角色（如前端开发者、测试工程师、项目经理）、分配任务给对应的 subagent 执行，并持续运行直至所有需求满足或任务被阻塞等待用户输入。

整个编排过程遵循一个目标对齐循环机制：每完成 5 个任务，系统会触发一次 backlog 重新检查，基于已完成的工作和发现的阻塞点，动态调整后续任务的优先级和方向。这种设计确保了 agent 系统在长时间自主运行中不会偏离初始目标，同时保留了灵活应变的能力。

对于用户交互，Skillwave 采用 `INBOX.md` 文件作为异步沟通通道。当 subagent 执行任务时遇到需要用户决策的问题（如技术选型、需求澄清），会将问题写入 INBOX.md 而非中断执行流程。用户可以在任意时刻查看并回复这些问题，系统则在后续执行中读取并处理这些反馈。这种异步模式避免了因等待用户输入而导致的全局阻塞，显著提升了系统的自主性和执行效率。

## 技术分析：五项创新设计点

### 一、异步用户沟通机制

传统 agent 系统通常通过实时对话获取用户输入，这导致系统在遇到决策点时必须暂停执行、等待用户响应。Skillwave 的 `INBOX.md` 设计将沟通异步化：agent 在执行过程中记录问题和待决策事项，用户可以批量处理这些问题而非逐一响应。这不仅提升了执行效率，也符合真实工作场景中"异步协作"的最佳实践。

### 二、持久真相源设计

`GOAL.md` 作为整个系统的目标锚点，提供了跨会话的一致性保障。当 agent 系统因环境重启、会话断开或其他原因重新启动时，GOAL.md 文件确保系统不会丢失初始目标，能够基于已有的 backlog 和完成记录继续执行。这种设计简单却关键——它让 agent 系统具备了真正的"长期记忆"，而非局限于单次对话上下文。

### 三、任务级 Git 提交

Skillwave 要求每个任务完成后立即进行 Git 提交，并在提交信息中关联 backlog 条目。这提供了完整的版本历史追溯能力：用户可以清晰看到每个任务对应的代码变更、理解系统的执行轨迹，并在必要时回滚到特定状态。相比于传统方案中"完成后一次性提交"的批量模式，任务级提交降低了风险、提升了可维护性。

### 四、定期目标对齐循环

每完成 5 个任务触发一次 backlog 检查，是 Skillwave 的核心编排策略。这种周期性对齐机制解决了 agent 系统的"目标漂移"问题：当执行过程中发现新的依赖、阻塞或更优方案时，系统并非盲目继续，而是基于已完成的工作重新审视剩余任务，调整优先级、修正方向。这模拟了真实项目管理中的"迭代规划"实践，赋予了系统动态应变能力。

### 五、无数据库依赖架构

Skillwave 的所有状态——包括用户故事 backlog、任务完成记录、用户问答、执行日志——均以 Markdown 文件形式存储在 Git 仓库中。这种设计有三个显著优势：架构简化、迁移成本低、版本控制天然集成。用户无需配置数据库、处理数据迁移脚本，只需克隆仓库即可在任何环境启动编排流程；所有状态变更自动进入 Git 历史，提供了完整的可追溯性。

## 总结：轻量编排范式的实践价值

Skillwave 的设计哲学可以概括为"简单而有效"：它没有引入复杂的状态管理框架、没有依赖外部数据库服务，而是充分利用 Markdown 文件的易编辑性、Git 的版本控制能力，构建了一个轻量、可维护、跨平台一致的 agent 编排系统。

作者的实践验证表明，这一方案在多个真实项目中取得了超预期的效果。其开源发布为技术社区提供了一个可参考的范式，也为 autonomous agent 系统的工程化落地提供了新的思路：在追求能力强大的同时，架构的简洁和可维护性同样关键，甚至在长期项目中更为重要。Skillwave 展示了如何用"对的工具做对的事"——以文件和版本控制构建持久状态，以异步沟通提升自主性，以周期对齐确保目标一致，为未来的 agent 编排系统设计提供了有价值的参考。
